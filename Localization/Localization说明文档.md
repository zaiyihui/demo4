# Localization 文件夹说明文档

## 1. 文件夹功能概述
**Localization** 文件夹用于存放项目的本地化资源文件，支持多语言切换，确保应用程序在不同语言环境下的正常显示。

## 2. 包含内容

| 文件/文件夹 | 文件类型 | 说明 | 重要性 |
|------------|---------|------|--------|
| **Strings.zh-CN.cs** | C# 源代码 | 简体中文本地化字符串资源 | ⭐⭐⭐⭐⭐ |

## 3. 重要文件说明

### Strings.zh-CN.cs
- **位置**：[Localization/Strings.zh-CN.cs](file:///d:/BC/xiangmu/demo4/demo4/Localization/Strings.zh-CN.cs)
- **功能**：存放简体中文环境下的所有用户界面字符串

#### 包含的字符串分类：
1. **通用文本**：按钮、菜单等通用 UI 元素
2. **主窗口**：主界面相关文本
3. **悬浮窗**：悬浮窗相关文本
4. **设置窗口**：设置界面相关文本
5. **错误提示**：错误信息和警告文本
6. **状态信息**：系统状态和提示信息
7. **单位**：百分比、温度等单位文本

## 4. 使用规范

### 4.1 字符串命名规范
- 使用 PascalCase 命名常量
- 使用描述性的名称，说明字符串用途
- 示例：`MainWindow_Title`、`Settings_SaveButton`

### 4.2 在代码中使用
```csharp
// 使用静态常量引用字符串
var title = Strings.zh_CN.MainWindow_Title;
```

### 4.3 添加新语言
如需添加新语言支持：
1. 创建新的字符串资源文件，如 `Strings.zh-TW.cs`（繁体中文）
2. 复制现有字符串结构
3. 翻译所有字符串
4. 更新项目配置以支持新语言

## 5. 本地化最佳实践

### 5.1 字符串设计
- 避免将 UI 文本硬编码在 XAML 或 C# 代码中
- 使用参数化的字符串，支持动态内容
- 考虑不同语言的文本长度差异

### 5.2 字符串示例
```csharp
// 硬编码（不推荐）
TextBlock.Text = "当前温度: 45°C";

// 参数化（推荐）
TextBlock.Text = string.Format(Strings.TemperatureFormat, 45);
```

### 5.3 编码规范
- 所有源文件使用 UTF-8 编码
- 在 .editorconfig 中配置编码规范
- 避免使用乱码字符

## 6. 维护建议

- 定期检查是否有硬编码的字符串需要提取
- 添加新功能时，同步更新本地化资源
- 保持不同语言版本的字符串一致性
- 考虑使用专业的本地化工具管理多语言资源
