# 🚨 F5调试启动失败 - 紧急修复指南

## 问题诊断

### ❌ 错误现象
按 F5 后无法成功启动调试

### 🔍 根本原因
**构建产物路径不匹配！**

launch.json 中的路径：
```json
"program": "${workspaceFolder}/bin/Debug/net8.0/win-x64/ComputerCompanion.dll"
```

实际构建产物在：
```
D:\BC\xiangmu\demo4\demo4\bin\Debug\net8.0\ComputerCompanion.dll
```

**注意**：实际路径没有 `win-x64` 子目录！

---

## ✅ 解决方案（2选1）

### 方案1️⃣：手动修复 launch.json（推荐）

#### 步骤1：打开文件
```
文件路径: d:\BC\xiangmu\demo4\demo4\.vscode\launch.json
```

#### 步骤2：修改路径
将所有的：
```json
"program": "${workspaceFolder}/bin/Debug/net8.0/win-x64/ComputerCompanion.dll"
```

替换为：
```json
"program": "${workspaceFolder}/bin/Debug/net8.0/ComputerCompanion.dll"
```

#### 步骤3：保存并测试
- 按 Ctrl+S 保存文件
- 按 F5 重新启动调试

## 🆘 如果仍然无法启动

### 问题1：仍然找不到文件
**解决方案**：
1. 在 VS Code 中打开终端：`Ctrl+`` `
2. 运行构建：
   ```bash
   dotnet build --configuration Debug
   ```
3. 检查是否有错误
4. 确保构建成功后再按 F5

### 问题2：编译错误
**解决方案**：
1. 查看终端中的红色错误信息
2. 修复所有编译错误
3. 重新构建
4. 再次按 F5

### 问题3：断点不触发
**解决方案**：
1. 确认是 Debug 配置（不是 Release）
2. 清理并重新构建：
   ```bash
   dotnet clean
   dotnet build --configuration Debug
   ```
3. 再次按 F5
