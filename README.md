# 电脑伴侣（Computer Companion）

## 项目概述

**电脑伴侣** 是一款基于 Avalonia UI 框架开发的跨平台硬件监控桌面应用程序，专注于为游戏玩家和电脑爱好者提供实时的系统性能监控。

### 核心特性

- ✅ **实时硬件监控**：CPU、GPU、内存、网络、磁盘、电池等硬件状态的实时监控
- ✅ **NVIDIA 风格悬浮窗**：游戏时显示关键性能指标（FPS、延迟、使用率等）
- ✅ **系统托盘集成**：支持最小化到系统托盘，后台持续监控
- ✅ **个性化配置**：丰富的自定义选项，包括界面布局、颜色主题、显示内容等
- ✅ **中文本地化**：完整的中文用户界面和本地化支持
- ✅ **自包含发布**：无需安装 .NET 运行时，独立运行，与其他工具无冲突

## 核心功能介绍

### 1. 主窗口
- 显示所有硬件的详细监控数据
- 支持垂直/水平布局切换
- 可自定义显示内容

### 2. 悬浮窗
- NVIDIA 风格的游戏悬浮窗
- 可选择显示 FPS、CPU、GPU、内存、延迟等
- 支持四个角落位置
- 透明度和字体大小可调节

### 3. 设置窗口
- 6 个功能面板：概览、显示设置、外观、显示内容、性能、启动选项
- 实时预览配置效果
- 一键恢复默认设置

### 4. 系统托盘
- 最小化到托盘
- 右键菜单快速操作
- 左键点击显示主窗口

## 使用指南

### 快速开始

#### 方式一：使用已发布版本（推荐）

1. 进入发布目录：`bin/Release/net8.0/win-x64/publish/`
2. 双击运行：`电脑伴侣.exe`
3. 无需安装 .NET 运行时！

#### 方式二：从源代码编译运行

1. 确保安装了 [.NET 8.0 SDK](https://dotnet.microsoft.com/download/dotnet/8.0)
2. 在项目根目录执行：
   ```powershell
   dotnet build
   dotnet run
   ```

### 基本使用流程

1. **启动程序**：运行 `电脑伴侣.exe`
2. **配置悬浮窗**：打开设置窗口，根据需要配置
3. **开始游戏**：悬浮窗会自动显示关键性能数据
4. **查看详情**：点击托盘图标或主窗口查看完整硬件数据

### 主要配置选项

| 配置分类 | 选项 |
|---------|------|
| 显示设置 | 悬浮窗开关、位置、置顶 |
| 外观设置 | 透明度、字体大小、颜色 |
| 显示内容 | FPS、CPU、GPU、内存、延迟 |
| 性能设置 | 刷新频率、游戏模式 |
| 启动设置 | 自动启动、启动最小化 |

### 技术栈

| 类别 | 技术 | 版本 |
|------|------|------|
| UI 框架 | Avalonia UI | 12.0.3 |
| 语言 | C# | .NET 8.0 |
| MVVM | CommunityToolkit.Mvvm | 8.4.1 |
| 硬件监控 | LibreHardwareMonitorLib | 0.9.6 |
| 序列化 | Newtonsoft.Json | 13.0.4 |
| 测试 | xUnit | - |

### 架构模式

项目采用 **MVVM（Model-View-ViewModel）** 架构：

- **Model**：数据模型和业务逻辑（Models/、Services/）
- **View**：用户界面（Views/、Styles/）
- **ViewModel**：视图模型和数据绑定（ViewModels/、Converters/）

## 注意事项

### 使用注意

1. **系统权限**：某些硬件监控功能可能需要管理员权限
2. **性能影响**：监控本身有极小的资源占用，建议游戏时使用游戏模式降低刷新频率
3. **杀毒软件**：首次运行可能被误报，添加信任即可
4. **与 Watt Toolkit 兼容**：使用自包含版本，不会产生冲突

### 开发注意

1. **编码规范**：所有文件使用 UTF-8 编码
2. **代码风格**：遵循项目的代码规范（.editorconfig）
3. **文档更新**：修改代码时同步更新相关文档
4. **测试覆盖**：核心功能添加对应的单元测试

### 常见问题

**Q: 运行时提示缺少 .NET 运行时？**
A: 请使用 `bin/Release/net8.0/win-x64/publish/` 下的自包含版本。

**Q: 悬浮窗不显示？**
A: 请检查设置中是否启用了悬浮窗，以及悬浮窗位置是否在屏幕范围内。

**Q: 如何开机自启动？**
A: 在设置窗口的"启动选项"面板中启用"自动启动"。

更多问题请参考 [维护指南.md](docs/维护指南.md)。
