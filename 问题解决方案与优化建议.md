## 2. 问题 1: LibreHardwareMonitorLib 依赖加载问题

### 2.1 问题描述
尽管 LibreHardwareMonitorLib.dll 文件物理存在，但.NET 但无法加载。

### 2.2 原因分析
可能的原因：
1. **DLL 版本不匹配
2. **架构不匹配
3. **运行时无法找到 DLL（加载上下文问题
4. **缺少依赖项
5. **DLL文件损坏
6. **LibreHardwareMonitorLib 有特殊的加载要求

### 2.3 解决方案

#### 方案 A：确保正确复制 DLL（推荐）
**步骤：**

```powershell
# 1. 清理旧的 DLL
Remove-Item -Path "bin\Debug\net8.0\LibreHardwareMonitorLib.dll" -Force -ErrorAction SilentlyContinue

# 2. 从 NuGet 包中复制正确的 DLL
$nugetPath = "$env:USERPROFILE\.nuget\packages\librehardwaremonitorlib\0.9.6\runtimes\win-x64\lib\net8.0\LibreHardwareMonitorLib.dll"

# 3. 复制到输出目录
Copy-Item -Path $nugetPath -Destination "bin\Debug\net8.0\" -Force

Write-Host "✅ 已成功复制 LibreHardwareMonitorLib.dll" -ForegroundColor Green
```

#### 方案 B：在 csproj 中确保依赖复制设置
修改 `ComputerCompanion.csproj`，添加：

```xml
<PropertyGroup>
    <CopyLocalLockFileAssemblies>true</CopyLocalLockFileAssemblies>
    <EnableDynamicLoading>true</EnableDynamicLoading>
</PropertyGroup>
```

#### 方案 C：修改代码层面处理
在应用启动时处理加载问题
创建启动引导程序：

```csharp
// 在 Program.cs 或 App.axaml.cs
// 添加程序集解析事件
using System.Reflection;
using System.IO;

// 在启动前添加：
AppDomain.CurrentDomain.AssemblyResolve += (sender, args) =>
{
    var assemblyName = new AssemblyName(args.Name);
    
    if (assemblyName.Name == "LibreHardwareMonitorLib")
    {
        try
        {
            // 尝试从多个位置查找
            var possiblePaths = new[]
            {
                Path.Combine(AppDomain.CurrentDomain.BaseDirectory, "LibreHardwareMonitorLib.dll"),
                Path.Combine(AppDomain.CurrentDomain.BaseDirectory, "runtimes", "win-x64", "native", "LibreHardwareMonitorLib.dll"),
                // 从 NuGet 包缓存
                Path.Combine(Environment.GetFolderPath(Environment.SpecialFolder.UserProfile), ".nuget", "packages", "LibreHardwareMonitorLib", "0.9.6", "runtimes", "win-x64", "lib", "net8.0", "LibreHardwareMonitorLib.dll")
            };
            
            foreach (var path in possiblePaths.Where(File.Exists))
            {
                if (File.Exists(path))
                {
                    Console.WriteLine($"从以下加载 LibreHardwareMonitorLib.dll: {path}");
                    return Assembly.LoadFrom(path);
                }
            }
        }
        catch (Exception ex)
        {
            Console.WriteLine($"加载 LibreHardwareMonitorLib 时出错: {ex.Message}");
        }
    }
    return null;
};
```

### 2.4 验证步骤

## 4. 问题 3: 管理员权限问题

### 4.1 为什么需要管理员权限
- **原因：
1. 访问底层硬件监控API
2. 读取 CPU/GPU 温度和性能计数器
3. 读取系统底层信息
4. 修改系统级访问硬件监控服务

### 4.2 解决方案

#### 方案 A：添加 app.manifest 请求管理员权限
在项目中添加 `app.manifest`：
```xml
<?xml version="1.0" encoding="utf-8"?>
<assembly manifestVersion="1.0" xmlns="urn:schemas-microsoft-com:asm.v1">
  <trustInfo xmlns="urn:schemas-microsoft-com:asm.v2">
    <security>
      <requestedPrivileges xmlns="urn:schemas-microsoft-com:asm.v3">
        <!-- UAC 清单选项
        <requestedExecutionLevel level="requireAdministrator" uiAccess="false" />
      </requestedPrivileges>
    </security>
  </trustInfo>
</assembly>
```

#### 方案 B：代码层面优雅处理权限缺失
在代码中检测权限并降级功能：
```csharp
using System.Security.Principal;

public static bool IsAdministrator()
{
    using (var identity = WindowsIdentity.GetCurrent())
    {
        var principal = new WindowsPrincipal(identity);
        return principal.IsInRole(WindowsBuiltInRole.Administrator);
    }
}

// 在启动时检测：
if (!IsAdministrator())
{
    Console.WriteLine("⚠️ 警告：未使用管理员权限运行！");
    Console.WriteLine("硬件监控部分功能可能受限");
}
```

#### 方案 C：提供用户提示启动器
创建 `Launcher.cs 项目，要求管理员权限：

```csharp
// 如果非管理员时，提示用户重启为管理员
public static void EnsureAdministratorIfNeeded()
{
    if (!IsAdministrator())
    {
        try
        {
            var proc = new ProcessStartInfo
            {
                UseShellExecute = true,
                FileName = Process.GetCurrentProcess().MainModule.FileName,
                Verb = "runas", // 请求管理员权限
            };
            Process.Start(proc);
            Environment.Exit(0);
        }
        catch
        {
            // 用户取消，以非权限运行
        }
    }
}
```

### 5.2 路径优化：移动项目到上层目录

#### 迁移脚本 `move-project.ps1：

```powershell
# 电脑伴侣项目路径迁移脚本
# 将项目从 demo4\demo4 移动到 demo4

param([switch]$Force)

$ErrorActionPreference = "Stop"

Write-Host "========================================" -ForegroundColor Cyan
Write-Host "电脑伴侣项目路径迁移工具" -ForegroundColor Cyan
Write-Host "========================================" -ForegroundColor Cyan
Write-Host ""

$sourceDir = "D:\BC\xiangmu\demo4\demo4
$targetDir = "D:\BC\xiangmu\demo4

Write-Host "源目录: $sourceDir" -ForegroundColor Green
Write-Host "目标目录: $targetDir" -ForegroundColor Green
Write-Host ""

# 检查源目录存在
if (-not (Test-Path $sourceDir))
{
    Write-Host "❌ 错误：源目录不存在！" -ForegroundColor Red
    exit 1
}

Write-Host "正在扫描源目录..." -ForegroundColor Yellow
$items = Get-ChildItem -Path $sourceDir -Force

Write-Host "找到 $($items.Count) 个项目" -ForegroundColor Green
Write-Host ""

Write-Host "开始移动文件..." -ForegroundColor Cyan

# 排除列表
$excludeItems = @("bin", "obj", ".vs", ".git")

$moved = 0
$skipped = 0
$errors = 0

foreach ($item in $items)
{
    # 跳过排除目录
    if ($item.PSIsContainer -and $excludeItems -contains $item.Name)
    {
        Write-Host "⏭️ 跳过: $($item.Name)" -ForegroundColor Gray
        $skipped++
        continue
    }

    $targetPath = Join-Path $targetDir $item.Name

    #检查目标已存在
    if (Test-Path $targetPath)
    {
        Write-Host "⚠️ 已存在，跳过: $($item.Name)" -ForegroundColor Yellow
        $skipped++
        continue
    }

    try
    {
        Write-Host "移动: $($item.Name)" -ForegroundColor Green
        Move-Item -Path $item.FullName -Destination $targetPath -Force
        $moved++
    }
    catch
    {
        Write-Host "❌ 失败: $($item.Name): $($_.Exception.Message)" -ForegroundColor Red
        $errors++
    }
}

Write-Host ""
Write-Host "========================================" -ForegroundColor Cyan
Write-Host "移动完成！" -ForegroundColor Cyan
Write-Host "========================================" -ForegroundColor Cyan
Write-Host "✅ 成功: $moved" -ForegroundColor Green
Write-Host "⏭️ 跳过: $skipped" -ForegroundColor Yellow
Write-Host "❌ 失败: $errors" -ForegroundColor Red
Write-Host ""

# 验证是否成功
if ((Test-Path (Join-Path $targetDir "ComputerCompanion.csproj"))
{
    Write-Host "🎉 项目成功到: $targetDir" -ForegroundColor Green
    Write-Host ""
    Write-Host "下一步:" -ForegroundColor Yellow
    Write-Host "1. 在新目录中打开项目"
    Write-Host "2. 更新: $targetDir"
    Write-Host "3. 重新构建项目"
    Write-Host ""
    Write-Host "4. 测试运行"
}
else
{
    Write-Host "⚠️ 警告：未找到项目文件" -ForegroundColor Yellow
}

Write-Host ""
Write-Host "按任意键退出..."
$Host.UI.RawUI.ReadKey("NoEcho,IncludeKeyDown") | Out-Null
```

## 6. 问题 5: 运行时问题排查

### 6.1 应用启动后立即退出
### 原因分析
1. **初始化异常被静默捕获
2. **窗口创建失败
3. **DLL加载失败
4. **配置文件问题

### 解决方案

#### 方案 A：添加详细启动日志
在 `Program.cs` 中增强：

```csharp
using System;
using System.IO;
using System.Diagnostics;

[STAThread]
public static void Main(string[] args)
{
    // 1. 设置日志文件
    var logPath = Path.Combine(
        Environment.GetFolderPath(Environment.SpecialFolder.LocalApplicationData),
        "ComputerCompanion",
        "startup.log");
    
    Directory.CreateDirectory(Path.GetDirectoryName(logPath));
    
    using (var logWriter = new StreamWriter(logPath, append: true))
    {
        logWriter.AutoFlush = true;
        logWriter.WriteLine($"[{DateTime.Now:yyyy-MM-dd HH:mm:ss} 应用程序启动");
        
        try
        {
            logWriter.WriteLine("开始初始化...");
            BuildAvaloniaApp().StartWithClassicDesktopLifetime(args);
            logWriter.WriteLine("应用程序正常退出");
        }
        catch (Exception ex)
        {
            logWriter.WriteLine($"❌ 应用程序崩溃: {ex}");
            Console.WriteLine($"错误: {ex.Message}");
            Console.WriteLine($"详细: {ex}");
            
            // 显示错误框
            if (!Debugger.IsAttached)
            {
                MessageBox.Show($"错误: {ex.Message}\n\n详细日志: {logPath}", 
                    "启动失败",
                    MessageBoxButtons.OK, MessageBoxIcon.Error);
            }
        }
    }
}
```

#### 方案 B：添加崩溃处理
在 `App.axaml.cs` 中：

```csharp
public override void OnFrameworkInitializationCompleted()
{
    try
    {
        // 现有初始化代码
        // ...
    }
    catch (Exception ex)
    {
        Console.WriteLine($"❌ 应用初始化: {ex}");
        
        // 显示错误
        if (ApplicationLifetime is IClassicDesktopStyleApplicationLifetime desktop)
        {
            // 错误时显示错误窗口
            var errorWindow = new Window
            {
                Title = "启动失败",
                Width = 400,
                Height = 300,
                Content = new ScrollViewer
                {
                    Content = new TextBlock
                    {
                        Text = $"应用启动失败:\n\n{ex.Message}\n\n堆栈:\n{ex.StackTrace}",
                        TextWrapping = TextWrapping.Wrap
                    }
                }
            };
            desktop.MainWindow = errorWindow;
        }
    }
}
```
