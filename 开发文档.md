# 电脑伴侣 - 开发文档

## 1. 项目概述

电脑伴侣是一款基于 Avalonia UI 的跨平台硬件监控工具，提供实时硬件监控、游戏性能显示和系统状态追踪功能。

### 技术栈

| 组件 | 版本 | 说明 |
|------|------|------|
| .NET | 8.0 | 运行时框架 |
| Avalonia UI | 11.0.10 | 跨平台 UI 框架 |
| CommunityToolkit.Mvvm | 8.4.1 | MVVM 工具包 |
| LibreHardwareMonitorLib | 0.9.6 | 硬件监控库 |

## 2. 架构设计

### 2.1 架构模式

采用经典的 MVVM（Model-View-ViewModel）架构模式：

```
┌─────────────────────────────────────────────────────────────┐
│                        View                               │
│  (MainWindow, SettingsWindow, OverlayWindow)              │
└──────────────────────────────┬────────────────────────────┘
                               │ 数据绑定
                               ▼
┌─────────────────────────────────────────────────────────────┐
│                    ViewModel                               │
│  (MainWindowViewModel, SettingsViewModel, OverlayViewModel)│
└──────────────────────────────┬────────────────────────────┘
                               │ 调用服务
                               ▼
┌─────────────────────────────────────────────────────────────┐
│                      Service                               │
│  (HardwareMonitorService, SettingsService, IpcService,     │
│   ColorPresetService, TrayIconService)                     │
└──────────────────────────────┬────────────────────────────┘
                               │ 数据模型
                               ▼
┌─────────────────────────────────────────────────────────────┐
│                       Model                                │
│  (Settings, MainWindowSettings, OverlaySettings,           │
│   ColorPreset, HardwareInfo)                              │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 模块划分

| 模块 | 职责 | 主要文件 |
|------|------|----------|
| Views | UI 界面展示 | Views/*.axaml, Views/*.axaml.cs |
| ViewModels | 业务逻辑处理 | ViewModels/*.cs |
| Services | 核心服务 | Services/*.cs |
| Models | 数据模型定义 | Models/*.cs |

## 3. 核心服务

### 3.1 HardwareMonitorService

**职责**: 实时硬件数据采集和监控

**主要功能**:
- CPU 使用率和温度监控
- GPU 使用率、温度和显存监控
- 内存使用监控
- 网络上传下载速度监控
- 磁盘空间监控
- 电池状态监控
- 网络延迟测量

**关键方法**:
- `Start(int intervalMs)` - 启动监控
- `Stop()` - 停止监控
- `MarkFrame()` - 标记帧用于 FPS 计算

### 3.2 SettingsService

**职责**: 配置管理

**主要功能**:
- 设置的加载和保存
- 配置文件备份和恢复
- 默认设置管理

**关键方法**:
- `GetSettings()` - 获取当前设置
- `SaveSettings()` - 保存设置
- `RestoreFromBackup()` - 从备份恢复
- `ResetToDefaults()` - 重置为默认值

### 3.3 IpcService

**职责**: 进程间通信

**主要功能**:
- 主程序与悬浮窗进程通信
- 消息发送和接收
- 自动重连机制

**关键方法**:
- `StartServerAsync()` - 启动服务器
- `ConnectAsync()` - 连接到服务器
- `SendMessageAsync(IpcMessage)` - 发送消息

### 3.4 ColorPresetService

**职责**: 颜色预设管理

**主要功能**:
- 预设的加载和保存
- 自定义预设的创建和删除
- 预设的导入和导出

**关键方法**:
- `GetAllPresets()` - 获取所有预设
- `AddPreset(ColorPreset)` - 添加预设
- `ExportPresets(string)` - 导出预设
- `ImportPresets(string)` - 导入预设

## 4. 数据模型

### 4.1 Settings

组合所有配置子模块的根配置类：

```csharp
public class Settings
{
    public MainWindowSettings MainWindow { get; set; }
    public OverlaySettings Overlay { get; set; }
    public DisplayContentSettings DisplayContent { get; set; }
    public PerformanceSettings Performance { get; set; }
    public StartupSettings Startup { get; set; }
}
```

### 4.2 子配置模块

| 模块 | 内容 |
|------|------|
| MainWindowSettings | 主窗口布局和显示设置 |
| OverlaySettings | 悬浮窗显示设置 |
| DisplayContentSettings | 显示内容开关 |
| PerformanceSettings | 性能监控设置 |
| StartupSettings | 启动行为设置 |

## 5. 启动流程

### 5.1 程序入口

```csharp
// Program.cs
public static void Main(string[] args)
{
    // 设置编码和文化
    // 检查启动参数
    App.IsOverlayMode = args.Contains("--overlay");
    BuildAvaloniaApp().StartWithClassicDesktopLifetime(args);
}
```

### 5.2 应用初始化

```csharp
// App.axaml.cs
public override void OnFrameworkInitializationCompleted()
{
    // 初始化服务
    _settingsService = new SettingsService();
    _hardwareMonitorService = new HardwareMonitorService();
    _hardwareMonitorService.Start();
    
    if (IsOverlayMode)
        InitializeOverlayMode(settings);
    else
        InitializeMainMode(settings);
}
```

## 7. 配置管理

### 7.1 配置文件位置

```
%APPDATA%\ComputerCompanion\settings.json
```

### 7.2 备份机制

系统会自动创建配置备份：
- `settings.json.backup` - 最新备份
- `settings.json.old` - 旧备份

### 7.3 向后兼容性

Settings 类保留了已废弃的属性以支持旧版本配置文件的迁移。建议新代码使用子模块属性：

```csharp
// 推荐方式（新版）
settings.MainWindow.LayoutMode = LayoutMode.Vertical;

// 兼容方式（旧版，已废弃）
settings.LayoutMode = LayoutMode.Vertical;
```

## 9. 常见问题

### 9.1 SkiaSharp 原生库缺失

**问题**: 启动时提示无法加载 libSkiaSharp.dll

**解决方案**:
1. 运行 `快速启动.bat`，脚本会自动复制库文件
2. 或手动复制：
   ```
   %USERPROFILE%\.nuget\packages\skiasharp.nativeassets.win32\3.119.4-preview.1.1\runtimes\win-x64\native\libSkiaSharp.dll
   ```
   到 `bin/Debug/net8.0/` 目录

### 9.2 硬件监控数据为空

**问题**: 启动后显示 "--" 没有数据

**解决方案**:
- 确保以管理员权限运行
- 检查 LibreHardwareMonitorLib 是否正确加载

### 9.3 悬浮窗不显示

**问题**: 启用悬浮窗后不显示

**解决方案**:
- 检查设置中的悬浮窗位置是否在屏幕范围内
- 检查是否有其他程序遮挡
