
## 🔧 方法二：手动移动文件

### 步骤 1：关闭所有程序
- 关闭 VS Code
- 关闭任何正在运行的 ComputerCompanion 应用程序
- 关闭所有打开的终端窗口

### 步骤 2：删除旧的构建产物（可选但推荐）
在文件资源管理器中，删除以下目录（如果存在）：
- `d:\BC\xiangmu\demo4\demo4\bin`
- `d:\BC\xiangmu\demo4\demo4\obj`
- `d:\BC\xiangmu\demo4\demo4\.vs`

### 步骤 3：移动文件
1. 打开文件资源管理器
2. 导航到 `d:\BC\xiangmu\demo4\demo4`
3. **全选所有文件和文件夹**（除了 bin、obj、.vs）
4. **剪切**（Ctrl + X）
5. 导航到 `d:\BC\xiangmu\demo4`
6. **粘贴**（Ctrl + V）

### 步骤 4：验证
确保以下文件在 `d:\BC\xiangmu\demo4` 目录中：
- ✅ `ComputerCompanion.csproj`
- ✅ `Program.cs`

- ✅ `README.md`

## 🔧 方法三：使用 PowerShell 命令

如果您想直接使用 PowerShell 命令，请按以下步骤操作：

```powershell
# 1. 以管理员身份打开 PowerShell
# 2. 执行以下命令：

# 设置路径
$source = "d:\BC\xiangmu\demo4\demo4"
$target = "d:\BC\xiangmu\demo4"

# 要排除的目录
$exclude = @("bin", "obj", ".vs", ".git")

# 获取所有项目
Get-ChildItem -Path $source -Force | Where-Object { 
    $_.Name -notin $exclude 
} | ForEach-Object {
    $dest = Join-Path $target $_.Name
    if (-not (Test-Path $dest)) {
        Move-Item -Path $_.FullName -Destination $dest -Force
        Write-Host "Moved: $($_.Name)"
    } else {
        Write-Host "Skipped (exists): $($_.Name)"
    }
}
```

## ⚠️ 常见问题

### Q: 移动时提示"文件正在使用"怎么办？
A: 确保关闭以下程序：
- VS Code
- 任何正在运行的 ComputerCompanion 应用程序
- 所有终端/命令提示符窗口
- 任何可能打开项目文件的程序

### Q: 移动后无法运行项目怎么办？
A: 请执行以下步骤：
1. 删除 `bin` 和 `obj` 文件夹
2. 运行 `dotnet restore`
3. 运行 `dotnet build`
4. 然后再运行项目

### Q: VS Code 调试不工作怎么办？
A: 
1. 检查 `.vscode/launch.json` 中的路径是否正确
2. 如果需要，请使用之前提供的模板重新创建配置
3. 确保工作目录是 `d:\BC\xiangmu\demo4`
