## 1. 目录结构概览

```

│   ├── MainWindow.axaml(.cs)      # 主窗口视图

│   ├── 部署运维指南.md
│   ├── 项目结构说明.md

│       ├── 硬件监控服务.md
```

## 2. 目录功能说明

### 2.1 源代码目录

| 目录 | 功能描述 | 包含文件类型 |
|------|----------|--------------|
| **Models** | 数据模型层，定义业务数据结构和枚举 | `.cs` |
| **Services** | 核心服务层，提供应用业务逻辑和数据处理 | `.cs` |
| **ViewModels** | 视图模型层，连接视图与业务逻辑 | `.cs` |
| **Views** | 视图层，定义用户界面 | `.axaml`, `.axaml.cs` |
| **Assets** | 静态资源，如图标、图片等 | `.ico`, `.png` |

### 2.2 配置目录

| 目录 | 功能描述 |
|------|----------|
| **.vscode** | VS Code 编辑器配置，包括调试和任务配置 |

### 2.3 文档目录

| 目录 | 功能描述 |
|------|----------|
| **docs** | 项目文档，包含概况、部署指南、模块说明等 |
| **docs/模块文档** | 各功能模块的详细技术文档 |

## 3. 文件组织原则

### 3.1 命名规范

| 类型 | 命名规则 | 示例 |
|------|----------|------|
| **类文件** | PascalCase | `MainWindowViewModel.cs` |
| **视图文件** | 窗口名 + 扩展名 | `MainWindow.axaml` |
| **代码后端** | 视图名 + .cs | `MainWindow.axaml.cs` |
| **资源文件** | 小写下划线或 PascalCase | `avalonia-logo.ico` |

### 3.2 文件引用规范

| 引用类型 | 使用方式 |
|----------|----------|
| **同目录引用** | 直接文件名 |
| **跨目录引用** | 完整命名空间 |
| **服务引用** | 通过依赖注入 |

## 5. 特殊文件说明

| 文件 | 说明 |
|------|------|
| **App.axaml** | Avalonia 应用资源定义，包含样式和主题 |
| **App.axaml.cs** | 应用生命周期管理 |
| **Program.cs** | 程序入口点，处理命令行参数 |
| **ViewLocator.cs** | MVVM 视图定位器，根据 ViewModel 查找对应视图 |
| **app.manifest** | Windows 应用清单，定义兼容性和权限 |
| **.gitignore** | Git 版本控制忽略规则 |

## 7. 新增文件规范

### 7.1 添加新模型

```
Models/ 新文件名.cs
```

### 7.2 添加新服务

```
Services/ 新服务名Service.cs
```

### 7.3 添加新视图

```
Views/ 新窗口名.axaml      # XAML 定义
Views/ 新窗口名.axaml.cs    # 代码后端
```

### 7.4 添加新视图模型

```
ViewModels/ 新视图模型名ViewModel.cs
```

## 8. 目录维护准则

| 准则 | 说明 |
|------|------|
| **高内聚** | 相关功能文件应放在同一目录 |
| **低耦合** | 避免跨多目录的复杂引用 |
| **一致性** | 同类文件使用统一的命名风格 |
| **可发现性** | 目录和文件名应清晰表达功能 |
| **最小化** | 只保留必要的文件和目录 |
