# Computer Companion 部署运维文档

---

## 1. 环境配置要求

### 1.1 开发环境

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11 (推荐) |
| .NET SDK | .NET 8.0 或更高版本 |
| IDE | Visual Studio 2022 / Rider / VS Code |
| 构建工具 | dotnet CLI |

### 1.2 运行环境

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11 (x64) |
| .NET Runtime | .NET 8.0 Desktop Runtime |
| 权限 | 普通用户权限（监控硬件需要管理员权限） |

## 2. 开发环境搭建

### 2.1 安装 .NET SDK

1. 下载 .NET 8.0 SDK：https://dotnet.microsoft.com/download/dotnet/8.0
2. 运行安装程序，选择 "Desktop development" 工作负载
3. 验证安装：

```bash
dotnet --version
# 输出示例: 8.0.100
```

### 2.2 克隆项目

```bash
git clone <repository-url>
cd ComputerCompanion
```

### 2.3 安装依赖

```bash
dotnet restore
```

### 2.4 构建项目

```bash
# Debug 构建
dotnet build -c Debug

# Release 构建
dotnet build -c Release
```

### 2.5 运行项目

```bash
# 运行主程序
dotnet run -c Debug

# 运行悬浮窗模式
dotnet run -c Debug -- --overlay
```

## 3. 生产环境部署

### 3.1 发布应用

```bash
# 发布自包含版本（推荐）
dotnet publish -c Release -r win-x64 --self-contained true /p:PublishSingleFile=true

# 发布框架依赖版本
dotnet publish -c Release -r win-x64
```

### 3.2 发布输出结构

```
bin/Release/net8.0/win-x64/publish/
├── ComputerCompanion.exe    # 主程序可执行文件
├── ComputerCompanion.pdb    # 调试符号文件
├── ComputerCompanion.deps.json
├── ComputerCompanion.runtimeconfig.json
└── 各种依赖 DLL 文件
```

### 3.3 部署步骤

1. 创建部署目录：
```bash
mkdir C:\Program Files\ComputerCompanion
```

2. 复制发布文件到部署目录

3. 创建快捷方式（可选）：
- 右键点击 `ComputerCompanion.exe`
- 选择 "创建快捷方式"
- 将快捷方式移动到桌面或开始菜单

### 3.4 开机自启配置

应用支持通过设置界面开启开机自启功能，会自动在以下注册表位置添加启动项：

```
HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\Run\电脑伴侣
```

## 4. 配置文件管理

### 4.1 配置文件位置

```
%APPDATA%\ComputerCompanion\settings.json
```

### 4.2 配置项说明

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `LayoutMode` | string | Vertical | 布局模式 |
| `TextColor` | string | #FFFFFF | 文字颜色 |
| `BackgroundColor` | string | #1a1a2eea | 背景颜色 |
| `BackgroundOpacity` | double | 0.9 | 背景透明度 |
| `FontSize` | int | 14 | 字体大小 |
| `RefreshInterval` | int | 1000 | 刷新间隔(ms) |
| `GameMode` | bool | false | 游戏模式 |
| `EnableOverlay` | bool | true | 启用悬浮窗 |
| `OverlayAlwaysOnTop` | bool | true | 悬浮窗置顶 |
| `AutoStart` | bool | false | 开机自启 |
| `StartMinimized` | bool | false | 启动时最小化 |

### 4.3 配置文件备份

建议定期备份配置文件：

```bash
copy "%APPDATA%\ComputerCompanion\settings.json" "%APPDATA%\ComputerCompanion\settings.json.bak"
```

## 5. 监控与日志

### 5.1 日志输出

应用通过 `Console.WriteLine` 输出日志信息，主要包含：

| 日志类型 | 内容 |
|----------|------|
| 启动日志 | 服务初始化信息 |
| 错误日志 | 异常信息和错误堆栈 |
| 警告日志 | 警告信息 |
| 调试日志 | 调试信息 |

### 5.2 性能监控

应用内置性能监控功能，可监控：

- CPU 使用率和温度
- GPU 使用率和温度
- 内存使用情况
- 网络上传/下载速率
- 磁盘空间
- 电池电量

### 5.3 常见问题排查

| 问题 | 可能原因 | 解决方法 |
|------|----------|----------|
| 硬件信息无法获取 | 缺少管理员权限 | 以管理员身份运行 |
| 悬浮窗不显示 | 被其他窗口遮挡 | 检查置顶设置 |
| 网络速率显示负数 | 网络接口数据重置 | 已在代码中修复 |
| 应用无法启动 | 缺少 .NET Runtime | 安装 .NET 8.0 Desktop Runtime |

## 6. 故障处理

### 6.1 应用崩溃

1. **查看事件查看器**：
   - 打开 "事件查看器"
   - 导航到 "Windows 日志" > "应用程序"
   - 查找来源为 ".NET Runtime" 的错误事件

2. **检查日志文件**：
   - 检查应用目录下是否有崩溃日志
   - 查看 Windows 事件日志中的详细错误信息

### 6.2 硬件监控失败

| 错误现象 | 处理步骤 |
|----------|----------|
| CPU 信息为空 | 确保 LibreHardwareMonitorLib 正确加载 |
| GPU 信息为空 | 确保显卡驱动已正确安装 |
| 网络信息异常 | 检查网络适配器状态 |
| 电池信息异常 | 仅支持 Windows 平台 |

### 6.3 悬浮窗无法启动

1. 检查是否已启用悬浮窗：
   ```json
   "EnableOverlay": true
   ```

2. 检查悬浮窗进程是否已启动：
   ```bash
   tasklist | findstr ComputerCompanion
   ```

3. 重启应用尝试重新启动悬浮窗进程

## 7. 版本更新

### 7.1 更新流程

1. **备份配置文件**：
   ```bash
   copy "%APPDATA%\ComputerCompanion\settings.json" "%APPDATA%\ComputerCompanion\settings.json.bak"
   ```

2. **停止应用**：
   - 关闭主程序窗口
   - 确保悬浮窗进程已终止

3. **替换文件**：
   - 停止应用
   - 复制新版本文件到部署目录

4. **启动应用**：
   - 运行 `ComputerCompanion.exe`
   - 验证配置是否正确加载

### 7.2 版本兼容性

| 版本 | 配置兼容性 | 说明 |
|------|------------|------|
| v1.0+ | 完全兼容 | 配置文件格式一致 |

## 8. 安全注意事项

### 8.1 权限管理

- 硬件监控需要管理员权限才能获取完整信息
- 建议以普通用户权限运行，仅在需要时提升权限

### 8.2 数据安全

- 配置文件存储在用户目录，不包含敏感信息
- 不收集用户个人数据
- 不向外部服务器发送数据

### 8.3 防病毒软件

部分防病毒软件可能会误报硬件监控行为，建议：
- 将应用目录添加到白名单
- 使用数字签名（生产环境）
