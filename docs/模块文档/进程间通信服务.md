# IPC 通信服务模块文档

---

## 1. 模块概述

### 1.1 功能描述

IPC（进程间通信）服务负责主程序和悬浮窗进程之间的通信，支持以下功能：

- **设置同步**：主程序设置变更后同步到悬浮窗
- **窗口控制**：控制主窗口显示/隐藏
- **应用退出**：通知悬浮窗进程退出

### 1.2 设计目标

| 目标 | 说明 |
|------|------|
| **可靠性** | 支持自动重连，确保通信稳定 |
| **异步通信** | 非阻塞通信，不影响 UI 响应 |
| **简单协议** | 基于 JSON 的消息格式，易于扩展 |
| **双向通信** | 支持服务器和客户端双向消息传递 |

### 2.1 架构设计

```
┌─────────────────────────────────────────────────────────────────┐
│                        IPC 通信架构                            │
├─────────────────────────────────────────────────────────────────┤
│                                                               │

│   │  (Server Mode)  │                                 │  (Client Mode)   │
│   └────────┬────────┘                                 └────────┬────────┘
│            │                                                  │
│            ▼                                                  ▼
│   ┌─────────────────┐                              ┌─────────────────┐
│   │  IpcService     │                              │  IpcService     │
│   │  (Server)       │                              │  (Client)       │
│   └────────┬────────┘                              └────────┬────────┘
│            │                                                  │
│            ▼                                                  ▼
│   ┌─────────────────┐                              ┌─────────────────┐
│   │   App           │                              │   App           │
```

### 2.2 核心组件

| 组件 | 职责 | 实现方式 |
|------|------|----------|
| **NamedPipeServerStream** | 服务器端管道 | .NET System.IO.Pipes |

| **CancellationTokenSource** | 取消令牌 | 控制异步操作取消 |

## 4. 数据流程

### 4.1 服务器启动流程

```
StartServerAsync()
    │
    ├─→ 设置 _isServer = true
    │
    ├─→ 创建 CancellationTokenSource
    │
    └─→ 进入服务器循环
          ├─→ 创建 NamedPipeServerStream
          ├─→ 等待客户端连接 (WaitForConnectionAsync)
          ├─→ 触发 Connected 事件
          └─→ 启动消息读取任务
```

### 4.2 客户端连接流程

```
ConnectAsync()
    │
    ├─→ 设置 _isServer = false
    │
    ├─→ 创建 CancellationTokenSource
    │
    └─→ 进入客户端循环
          ├─→ 创建 NamedPipeClientStream
          ├─→ 连接到服务器 (ConnectAsync)
          ├─→ 触发 Connected 事件
          └─→ 启动消息读取任务
```

### 4.3 消息发送流程

```
SendMessageAsync(message)
    │
    ├─→ 参数验证
    │
    ├─→ 获取管道流 (Server 或 Client)
    │
    ├─→ 检查连接状态
    │
    ├─→ JSON 序列化消息
    │
    ├─→ 发送消息长度 (4 字节)
    │
    ├─→ 发送消息内容
    │
    └─→ 刷新管道
```

### 4.4 消息接收流程

```
ReadMessagesLoopAsync()
    │
    ├─→ 循环读取消息
    │
    ├─→ 读取消息长度 (4 字节)
    │
    ├─→ 验证消息长度范围 (0-65536)
    │
    ├─→ 读取消息内容
    │
    ├─→ JSON 反序列化
    │
    └─→ 触发 MessageReceived 事件
```

## 5. 关键实现细节

### 5.1 消息协议

消息采用长度前缀格式：

```
┌─────────────────────────────────────┐
│  消息长度 (4 字节, Little-Endian)   │
│  消息内容 (JSON 字符串, UTF-8)     │
└─────────────────────────────────────┘
```

**设计要点**：
- 长度前缀确保消息边界正确识别
- UTF-8 编码支持中文等特殊字符

### 5.2 重连机制

```csharp
private async Task RunClientLoopAsync(CancellationToken cancellationToken)
{
    while (!cancellationToken.IsCancellationRequested && !_isDisposed)
    {
        try
        {
            _client?.Dispose();
            _client = new NamedPipeClientStream(".", PipeName, PipeDirection.InOut, PipeOptions.Asynchronous);
            
            using var connectCts = CancellationTokenSource.CreateLinkedTokenSource(cancellationToken);
            connectCts.CancelAfter(ConnectTimeoutMs);
            
            await _client.ConnectAsync(connectCts.Token);
            OnConnected();
            
            await ReadMessagesLoopAsync(_client, cancellationToken);
        }
        catch (OperationCanceledException)
        {
            if (!cancellationToken.IsCancellationRequested)
            {
                Console.WriteLine("IPC connection timeout, retrying...");
            }
        }
        catch (Exception ex)
        {
            Console.WriteLine($"IPC Client error: {ex.Message}");
        }
        
        if (!cancellationToken.IsCancellationRequested && !_isDisposed)
        {
            await Task.Delay(ReconnectDelayMs, cancellationToken);
        }
    }
}
```

**设计要点**：
- 连接失败后自动重试
- 重试间隔可配置（默认 2000ms）
- 连接超时可配置（默认 5000ms）

### 5.3 消息安全

```csharp
private async Task ProcessMessageAsync(byte[] buffer, int bytesRead)
{
    try
    {
        var json = Encoding.UTF8.GetString(buffer, 0, bytesRead);
        var message = JsonConvert.DeserializeObject<IpcMessage>(json);
        
        if (message != null && !string.IsNullOrEmpty(message.Type))
        {
            await Task.Run(() => MessageReceived?.Invoke(message));
        }
    }
    catch (Exception ex)
    {
        Console.WriteLine($"Failed to process IPC message: {ex.Message}");
    }
}
```

**设计要点**：
- 验证消息有效性
- 使用 `Task.Run` 避免阻塞消息读取循环
- 异常处理防止单个消息处理失败影响整体

## 6. 错误处理

### 6.1 异常处理策略

| 异常场景 | 处理方式 | 恢复策略 |
|----------|----------|----------|
| 连接超时 | 记录日志 | 自动重试连接 |
| 连接断开 | 触发 Disconnected 事件 | 自动重连 |
| 消息格式错误 | 记录日志，丢弃消息 | 继续处理后续消息 |
| 管道错误 | 记录日志 | 重启管道 |

### 6.2 边界情况处理

| 边界情况 | 处理方式 |
|----------|----------|
| 消息长度为 0 | 忽略，继续读取 |
| 消息长度过大 (>65536) | 记录日志，跳过 |
| JSON 解析失败 | 记录日志，丢弃消息 |
| 空消息类型 | 忽略，不触发事件 |
