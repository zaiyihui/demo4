# 硬件监控服务模块文档

---

## 1. 模块概述

### 1.1 功能描述

硬件监控服务（`HardwareMonitorService`）是 Computer Companion 的核心服务模块，负责实时采集和管理系统硬件信息，包括：

- **CPU 监控**：使用率、温度、风扇转速
- **GPU 监控**：使用率、温度、风扇转速、显存使用、时钟频率
- **内存监控**：已用内存、总内存
- **网络监控**：上传速率、下载速率、网络延迟
- **磁盘监控**：可用空间、总空间
- **电池监控**：电量百分比、充电状态
- **FPS 测量**：实时帧率统计

### 1.2 设计目标

| 目标 | 说明 |
|------|------|
| **实时性** | 默认 1 秒刷新间隔，确保数据实时更新 |
| **准确性** | 使用专业硬件监控库获取精确数据 |
| **稳定性** | 完善的异常处理和资源管理 |
| **可扩展性** | 模块化设计，便于添加新的监控项 |

## 4. 数据流程

### 4.1 启动流程

```
Start() 调用
    │
    ├─→ 初始化 Computer 对象
    │     ├─→ IsCpuEnabled = true
    │     ├─→ IsGpuEnabled = true
    │     ├─→ IsMemoryEnabled = true
    │     ├─→ IsStorageEnabled = true
    │     ├─→ IsNetworkEnabled = true
    │     └─→ IsBatteryEnabled = true
    │
    ├─→ 调用 Computer.Open() 打开硬件访问
    │
    ├─→ 创建 Timer，设置间隔
    │
    └─→ 启动网络延迟测量
          └─→ StartPingMonitor()
```

### 4.2 数据更新流程

```
Timer.Elapsed 触发
    │
    ▼
UpdateData()
    │
    ├─→ UpdateHardwareData()
    │     │
    │     ├─→ 遍历 Computer.Hardware
    │     ├─→ 调用 hardware.Update()
    │     └─→ ProcessHardware(hardware)
    │           │
    │           ├─→ ProcessSensor(sensor, hardwareType)
    │           │     ├─→ ProcessLoadSensor()
    │           │     ├─→ ProcessTemperatureSensor()
    │           │     ├─→ ProcessFanSensor()
    │           │     ├─→ ProcessDataSensor()
    │           │     ├─→ ProcessPowerSensor()
    │           │     └─→ ProcessClockSensor()
    │           │
    │           └─→ 递归处理 SubHardware
    │
    ├─→ UpdateNetworkData()
    │     ├─→ 获取所有网络接口
    │     ├─→ 累加 BytesReceived/BytesSent
    │     └─→ 计算速率 = (当前值 - 上次值) / 时间间隔
    │
    ├─→ UpdateDiskData()
    │     └─→ 获取第一个固定磁盘信息
    │
    ├─→ UpdateBatteryData()
    │     └─→ 调用 Win32 API GetSystemPowerStatus()
    │
    └─→ DataUpdated?.Invoke()
          └─→ ViewModel 响应事件更新 UI
```

### 4.3 FPS 测量流程

```
MarkFrame() 调用
    │
    ├─→ 首次调用?
    │     └─→ 记录初始时间戳，返回
    │
    ├─→ 帧计数 +1
    │
    └─→ 时间间隔 >= 1秒?
          └─→ 计算 FPS = 帧数 / 时间间隔
              └─→ 更新 Fps 属性
```

### 4.4 网络延迟测量流程

```
MeasureLatencyAsync()
    │
    ├─→ 遍历 ping 主机列表
    │     ├─→ 223.5.5.5 (阿里 DNS)
    │     ├─→ 119.29.29.29 (腾讯 DNS)
    │     ├─→ 8.8.8.8 (Google DNS)
    │     └─→ 1.1.1.1 (Cloudflare DNS)
    │
    ├─→ 发送 Ping 请求 (超时 1 秒)
    │
    ├─→ Ping 成功?
    │     └─→ 更新 NetworkLatency，返回
    │
    └─→ 所有主机失败?
          └─→ 保持上次值或设为 null
```

## 5. 关键实现细节

### 5.1 传感器数据处理

```csharp
private void ProcessSensor(ISensor sensor, HardwareType hardwareType)
{
    switch (sensor.SensorType)
    {
        case SensorType.Load:
            ProcessLoadSensor(sensor, hardwareType);
            break;
        case SensorType.Temperature:
            ProcessTemperatureSensor(sensor, hardwareType);
            break;
        // ... 其他传感器类型
    }
}
```

**设计要点**：
- 使用 `SensorType` 枚举区分不同类型的传感器
- 根据 `HardwareType` 区分 CPU、GPU、内存等硬件
- 使用传感器名称精确匹配（如 "CPU Total"、"GPU Core"）

### 5.2 网络速率计算

```csharp
private void UpdateNetworkData()
{
    // 获取当前统计值
    long totalBytesReceived = 0;
    long totalBytesSent = 0;
    
    foreach (var iface in NetworkInterface.GetAllNetworkInterfaces()
        .Where(n => n.OperationalStatus == OperationalStatus.Up && 
                   !n.Description.Contains("Loopback")))
    {
        var stats = iface.GetIPStatistics();
        totalBytesReceived += stats.BytesReceived;
        totalBytesSent += stats.BytesSent;
    }
    
    // 计算速率
    var elapsed = (DateTime.Now - _lastNetworkUpdate).TotalSeconds;
    if (elapsed > 0 && _lastBytesReceived > 0)
    {
        var downloadDiff = totalBytesReceived - _lastBytesReceived;
        var uploadDiff = totalBytesSent - _lastBytesSent;
        
        // 处理负数（网络接口重置时可能出现）
        if (downloadDiff >= 0 && uploadDiff >= 0)
        {
            NetworkDownload = Math.Max(0, (float)(downloadDiff / elapsed / 1024 / 1024));
            NetworkUpload = Math.Max(0, (float)(uploadDiff / elapsed / 1024 / 1024));
        }
    }
    
    // 保存当前值供下次计算
    _lastBytesReceived = totalBytesReceived;
    _lastBytesSent = totalBytesSent;
    _lastNetworkUpdate = DateTime.Now;
}
```

**设计要点**：
- 过滤掉 Loopback 接口
- 累加所有活跃接口的数据
- 处理网络接口统计重置导致的负数问题
- 使用 `Math.Max(0, ...)` 确保非负值

### 5.3 资源管理

```csharp
public void Dispose()
{
    Stop();
    GC.SuppressFinalize(this);
}

public void Stop()
{
    _isRunning = false;
    
    // 停止并释放定时器
    _dataTimer?.Stop();
    _dataTimer?.Dispose();
    _pingTimer?.Stop();
    _pingTimer?.Dispose();
    
    // 关闭硬件访问
    try
    {
        _computer?.Close();
    }
    catch (Exception ex)
    {
        Console.WriteLine($"关闭硬件监控失败: {ex.Message}");
    }
}
```

**设计要点**：
- 实现 `IDisposable` 接口
- 使用 `?.` 空值检查
- 异常处理避免资源释放失败影响程序退出

## 6. 错误处理

### 6.1 异常处理策略

| 异常场景 | 处理方式 | 恢复策略 |
|----------|----------|----------|
| 硬件访问失败 | 记录日志，继续运行 | 下次更新重试 |
| 网络不可用 | 记录日志，保留上次值 | 网络恢复后自动恢复 |
| 磁盘访问失败 | 记录日志，设为 null | 下次更新重试 |
| 电池信息不可用 | 记录日志，设为 null | 仅 Windows 支持 |

### 6.2 边界情况处理

| 边界情况 | 处理方式 |
|----------|----------|
| 无可用网络接口 | `NetworkUpload/Download` 设为 null |
| 无磁盘驱动器 | `DiskFreeSpace/TotalSpace` 设为 null |
| 无独立显卡 | `HasGpu` 返回 false，GPU 属性为 null |
| 台式机无电池 | `HasBattery` 返回 false，电池属性为 null |
| 网络速率负数 | 使用 `Math.Max(0, ...)` 确保非负 |

## 7. 性能优化

### 7.1 优化策略

| 优化项 | 实现方式 |
|--------|----------|
| **批量更新** | 定时器单次触发，一次性更新所有数据 |
| **延迟初始化** | FPS 测量首次调用时初始化 |
| **异步操作** | 网络延迟测量使用异步 Ping |
| **减少重复计算** | 缓存上次统计值，仅计算差值 |

### 7.2 性能指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 更新间隔 | 1000ms | 默认值，可配置 |
| 单次更新耗时 | < 50ms | 不阻塞 UI 线程 |
| 内存占用 | < 50MB | 稳定运行时 |
| CPU 占用 | < 1% | 正常监控时 |
