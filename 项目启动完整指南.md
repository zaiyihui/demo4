### 2.2 手动修复步骤

#### 步骤1: 修复 launch.json

**文件路径**: `d:\BC\xiangmu\demo4\demo4\.vscode\launch.json`

**问题**: 路径包含错误的 `win-x64` 目录

**修复方法**:

1. 在 VS Code 中打开 launch.json
2. 搜索 `win-x64`
3. 将所有：
   ```
   "program": "${workspaceFolder}/bin/Debug/net8.0/win-x64/ComputerCompanion.dll"
   ```
   替换为：
   ```
   "program": "${workspaceFolder}/bin/Debug/net8.0/ComputerCompanion.dll"
   ```
4. 保存文件 (Ctrl+S)

**参考文档**: [launch.json修复步骤.md](launch.json修复步骤.md)

## 五、常见问题与解决方案

### 5.1 问题：F5 报错 "找不到文件"

**原因**: launch.json 路径错误

**解决方案**:
1. 打开 launch.json
2. 删除所有 `win-x64/` 部分
3. 保存并重试

### 5.2 问题：编译错误

**解决方案**:
```powershell
dotnet clean
dotnet restore
dotnet build --configuration Debug
```

### 5.3 问题：应用启动后立即退出

**可能原因**:
- 缺少依赖的 DLL
- 权限不足
- 硬件监控服务异常

**解决方案**:
1. 以管理员权限运行
2. 检查事件查看器日志
3. 尝试重新构建项目

### 5.4 问题：悬浮窗不显示

**解决方案**:
1. 打开设置窗口
2. 检查"显示设置"中的"启用悬浮窗"
3. 确认悬浮窗位置在屏幕范围内
4. 尝试运行 `dotnet run -- --overlay`
