# 电脑伴侣 - 启动速度优化指南

## 问题描述

`dotnet run` 命令启动慢的原因：

1. **每次检查编译** - dotnet run 会检查代码是否变化
2. **依赖还原** - 每次可能检查 NuGet 包
3. **Avalonia UI 初始化** - UI 框架启动较慢
4. **SkiaSharp 加载** - 原生库加载耗时

---

## 解决方案

### 方案1：快速启动（推荐）✨

**文件**: [快速启动.bat](快速启动.bat)

**特点**:
- ✅ 跳过编译检查，直接运行
- ✅ 自动复制 SkiaSharp 原生库
- ✅ 启动速度最快（~2-5秒）

**使用方法**:
1. 双击 `快速启动.bat`
2. 等待应用程序启动

---

### 方案2：标准启动（完整构建）

**命令**:
```powershell
dotnet build --configuration Debug
dotnet run --no-build
```

**特点**:
- ✅ 确保代码已编译
- ✅ 启动速度较快
- ✅ 适合开发调试

---

### 方案3：发布版本（最稳定）

**命令**:
```powershell
dotnet publish --configuration Release --runtime win-x64 --self-contained true
bin\Release\net8.0\win-x64\publish\ComputerCompanion.exe
```

**特点**:
- ✅ 独立运行，无需 .NET 运行时
- ✅ 包含所有依赖
- ✅ 性能最优
- ⚠️ 首次发布需要几分钟

## SkiaSharp 原生库说明

### 问题
Avalonia UI 需要 SkiaSharp 原生库（libSkiaSharp.dll），但 NuGet 包不会自动复制到输出目录。

### 解决方案
快速启动脚本会自动复制正确版本（3.119.4）的原生库。

### 手动复制命令
```powershell
$skiaSource = "$env:USERPROFILE\.nuget\packages\skiasharp.nativeassets.win32\3.119.4-preview.1.1\runtimes\win-x64\native\libSkiaSharp.dll"
Copy-Item -Path $skiaSource -Destination "bin\Debug\net8.0\libSkiaSharp.dll" -Force
```

## 常见问题

### Q1：快速启动提示 SkiaSharp 未找到
**A**: 需要先运行一次完整构建来还原依赖：
```powershell
dotnet restore
dotnet build --configuration Debug
```

### Q2：启动后崩溃
**A**: SkiaSharp 版本不匹配。运行：
```powershell
快速启动.bat
```

### Q3：代码修改后快速启动没变化
**A**: 因为跳过了编译。需要先编译：
```powershell
dotnet build --configuration Debug
快速启动.bat
```
